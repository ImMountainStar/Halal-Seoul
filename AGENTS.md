# AGENTS.md

## 목적
`halal_seoul` 프로젝트에서 Codex가 과부하 없이, 앱 개발에 바로 도움되는 방식으로 작업하기 위한 최소 규칙.

## 프로젝트 컨텍스트
- 제품: 쇼핑몰 앱 + 할랄 AI 스캔 기능
- 우선순위: 쇼핑 UX 안정화 > 스캔 기능 고도화 (단, 통합은 유지)
- 주요 문서:
  - `README.md`
  - `docs/architecture_v1.md`
  - `docs/api_v1.md`
  - `docs/backend_structure_v1.md`

## 작업 원칙 (간단)
- 현재 요청 해결을 최우선으로 한다.
- **Simplicity First**: 요청하지 않은 기능은 추가하지 않는다.
- **Surgical Changes**: 요청된 범위의 파일/코드만 수정한다.
- 큰 재작성보다 작은 단위의 되돌릴 수 있는 변경을 우선한다.
- 동작이 바뀌면 코드와 문서를 함께 맞춘다.
- 모르면 추측하지 않는다. 로그/코드/문서를 직접 열어 근거를 확인한다.

## 구현 규칙
- 기존 폴더/파일 구조를 우선 유지한다.
- 수정 파일의 기존 스타일/네이밍을 따른다.
- 필요가 명확하지 않으면 의존성을 추가하지 않는다.
- 주석은 복잡한 로직에만 최소로 추가한다.

## 검증 규칙
- 변경 후 가능한 가장 작은 유효 검증을 수행한다. (스크립트/테스트/린트)
- 전체 검증이 어렵다면, 무엇을 실행 못했는지 명시한다.
- 데이터/룰 변경 시 간단한 전후 sanity check를 남긴다.

## 안전 규칙
- 명시적 요청 없이는 파괴적 명령을 실행하지 않는다.
  - 예: `rm -rf`, `git reset --hard`, 강제 checkout
- 사용자 관련 없는 변경은 되돌리지 않는다.
- 영향이 큰 리팩터링은 짧은 계획을 먼저 공유한다.

## 커뮤니케이션 규칙
- 답변은 짧고 실행 가능하게 작성한다.
- 항상 아래 3가지를 보고한다:
  1. 무엇을 바꿨는지
  2. 왜 바꿨는지
  3. 어떻게 검증했는지

## 기본 제외 항목
- 과도한 설계/추상화
- 요청 없는 대규모 아키텍처 개편
- 과한 규칙 추가
